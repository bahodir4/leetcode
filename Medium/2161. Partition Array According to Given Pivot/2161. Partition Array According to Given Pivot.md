# 2161. Partition Array According to Given Pivot

## Intuition

The problem requires:

1. All elements `< pivot` come first.
2. All elements `== pivot` come next.
3. All elements `> pivot` come last.
4. The relative order of elements in the `< pivot` group and the `> pivot` group must remain unchanged.

Since the ordering inside each group must be preserved, we can perform a **stable partition** by collecting elements into three separate lists and concatenating them at the end.

---

## Approach

Create three arrays:

- `less` → elements smaller than `pivot`
- `equal` → elements equal to `pivot`
- `greater` → elements larger than `pivot`

Traverse `nums` once:

- If `num < pivot`, append to `less`
- If `num == pivot`, append to `equal`
- Otherwise, append to `greater`

Finally return:

```python
less + equal + greater
```

Because elements are appended in the order they are encountered, their relative ordering is preserved automatically.

---

## Example

### Input

```python
nums = [9,12,5,10,14,3,10]
pivot = 10
```

### Build the groups

```python
less    = [9,5,3]
equal   = [10,10]
greater = [12,14]
```

### Result

```python
[9,5,3,10,10,12,14]
```

---

## Algorithm

1. Initialize three empty lists:
   - `less`
   - `equal`
   - `greater`
2. Iterate through `nums`.
3. Place each number into the appropriate list.
4. Return the concatenation of all three lists.

---

## Complexity Analysis

### Time Complexity

```text
O(n)
```

We traverse the array once.

### Space Complexity

```text
O(n)
```

Additional arrays are used to store all elements.

---

## Key Observation

The phrase:

> "The relative order of the elements less than pivot and the elements greater than pivot is maintained."

implies a **stable partition**.

Whenever a problem asks for:
- grouping elements into categories, and
- preserving original order,

using multiple lists (or queues) is often the simplest and most reliable solution.
