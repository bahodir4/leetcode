# 3689. Maximum Total Subarray Value I

## Intuition

The problem allows us to choose the **same subarray multiple times**.

Let:

```
value(subarray) = max(subarray) - min(subarray)
```

If the maximum value achievable by any single subarray is `best`, then choosing that same subarray `k` times gives:

```
k * best
```

Therefore, we only need to find the maximum possible value of a single subarray.

---

## Key Observation

The largest possible value of any subarray is:

```
max(nums) - min(nums)
```

Why?

- No subarray can have a maximum larger than the global maximum.
- No subarray can have a minimum smaller than the global minimum.
- The subarray between the positions of the global maximum and global minimum contains both values.

Therefore:

```
best = max(nums) - min(nums)
```

Since we can reuse the same subarray:

```
answer = k * best
```

---

## Algorithm

1. Find the global minimum element.
2. Find the global maximum element.
3. Compute:

   ```
   best = max(nums) - min(nums)
   ```

4. Return:

   ```
   k * best
   ```

---

## Correctness Proof

Let:

```
M = max(nums)
m = min(nums)
```

For any subarray:

```
value(subarray) = max(subarray) - min(subarray)
```

Since:

```
max(subarray) ≤ M
min(subarray) ≥ m
```

we have:

```
value(subarray) ≤ M - m
```

Thus, no subarray can have value greater than:

```
M - m
```

There exists a subarray containing both the global maximum and global minimum (the segment between their indices), whose value is exactly:

```
M - m
```

Therefore:

```
best = M - m
```

Since the same subarray may be chosen multiple times, choosing this optimal subarray `k` times yields:

```
k × (M - m)
```

which is the maximum possible total value.

Hence the algorithm is correct.

---

## Complexity Analysis

- Time Complexity: `O(n)`
- Space Complexity: `O(1)`

---

## Python Solution

```python
from typing import List

class Solution:
    def maxTotalValue(self, nums: List[int], k: int) -> int:
        return (max(nums) - min(nums)) * k
```
