# 2196. Create Binary Tree From Descriptions

## Intuition

Each description gives us a parent-child relationship:

```python
[parent, child, isLeft]
```

To build the tree:

1. Create a `TreeNode` for every unique value.
2. Connect each child to its parent using `left` or `right`.
3. Track all nodes that appear as children.
4. The root is the only node that never appears as a child.

---

## Approach

### Step 1: Create nodes

Use a dictionary:

```python
nodes[val] = TreeNode(val)
```

This allows us to quickly access any node.

---

### Step 2: Build relationships

For every description:

```python
parent, child, isLeft
```

- Create nodes if they do not already exist.
- If `isLeft == 1`, assign:

```python
parent.left = child
```

- Otherwise assign:

```python
parent.right = child
```

---

### Step 3: Find the root

Every child node has a parent.

Store all child values in a set:

```python
children.add(child)
```

After processing all descriptions:

- Any node value that is **not** in `children` must be the root.

---

## Example

Input:

```python
descriptions = [
    [20,15,1],
    [20,17,0],
    [50,20,1],
    [50,80,0],
    [80,19,1]
]
```

Tree construction:

```text
        50
       /  \
      20   80
     / \   /
    15 17 19
```

Children set:

```python
{15,17,20,80,19}
```

---

## Complexity Analysis

### Time Complexity

We process each description once:

```text
O(n)
```

where `n = len(descriptions)`.

---

### Space Complexity

We store:

- One node for each unique value.
- A set of child values.

```text
O(n)
```

---

## Key Observation

A binary tree has exactly one root, and the root is the **only node that never appears as a child**. Tracking all child nodes makes finding the root straightforward.
